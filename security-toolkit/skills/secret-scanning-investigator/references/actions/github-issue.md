# Action: Create GitHub Issue (8.7)

**Requires explicit confirmation before execution.**

## Confirmation Prompt

```
⚠️  CREATE GITHUB ISSUE

This will create a public/private issue in the repository with investigation findings.

Title: [Security] Secret Scanning Alert #{number} Investigation Report
Labels: security, secret-scanning

The issue will contain:
- Executive summary
- Timeline
- Risk assessment
- Recommended actions

Note: The actual secret value will NOT be included.

Type 'CONFIRM' to create issue or 'cancel':
```

## Execution

```bash
gh issue create --repo {owner}/{repo} \
  --title "[Security] Secret Scanning Alert #{number} Investigation Report" \
  --label "security,secret-scanning" \
  --body "{issue_body}"
```

## Issue Body Template

```markdown
## Secret Scanning Alert Investigation

**Alert:** #{number}
**Type:** {secret_type}
**Risk Rating:** {rating}
**Investigation Date:** {date}

---

### Executive Summary

{3-4 sentence summary of findings}

---

### Timeline

| Date | Event |
|------|-------|
| {date} | Secret introduced in commit `{sha}` |
| {date} | Alert opened by GitHub |
| {date} | Investigation completed |

---

### Risk Assessment

**Status:** {CONFIRMED ACTIVE / CONFIRMED INACTIVE / UNKNOWN}
**In Current Code:** {Yes/No}
**Exposure Window:** {N} days

---

### Recommended Actions

- [ ] {Action 1}
- [ ] {Action 2}
- [ ] {Action 3}

---

### Next Steps

{Guidance on what should happen next}

---

*Generated by Secret Scanning Investigator*
*Note: Secret value redacted for security*
```

## Privacy Considerations

The issue body should NEVER include:
- The actual secret value
- Full credentials or tokens
- Information that could aid exploitation

Always redact or use partial representations (e.g., `ghp_****...****abcd`).
